# 篮球拍球声音识别技术 - 技术架构文档

## 1. 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                        前端应用                              │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌──────────────┐  ┌─────────────────┐    │
│  │  UI 组件层   │  │  业务逻辑层   │  │   工具函数层     │    │
│  │  React组件   │  │  音频处理    │  │  Canvas绘图     │    │
│  │  Tailwind   │  │  状态管理     │  │  数学计算       │    │
│  └─────────────┘  └──────────────┘  └─────────────────┘    │
├─────────────────────────────────────────────────────────────┤
│                       Web APIs                              │
│  Web Audio API │ MediaDevices API │ Canvas API             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 技术栈

- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite
- **样式方案**：Tailwind CSS
- **状态管理**：Zustand
- **核心API**：Web Audio API, MediaDevices API, Canvas API

## 3. 路由定义

本项目为单页面应用，无需路由配置。

## 4. 核心模块设计

### 4.1 音频处理模块 (AudioProcessor)

**职责**：
- 管理AudioContext生命周期
- 配置AnalyserNode参数（FFT大小、采样率）
- 实时捕获音频数据
- 提供频率和时间域数据

**关键参数**：
- fftSize: 2048
- smoothingTimeConstant: 0.8
- minDecibels: -90
- maxDecibels: -10

### 4.2 拍球检测模块 (DribbleDetector)

**检测算法**：
- 基于短时能量检测
- 能量阈值判断
- 最小时间间隔限制（防止重复计数）

**关键参数**：
- energyThreshold: 128 (可调节)
- minInterval: 150ms
- decayRate: 0.95

### 4.3 可视化模块 (AudioVisualizer)

**职责**：
- 实时绘制波形图（时域信号）
- 绘制频谱分析图（频域信号）
- 动画循环控制

**性能优化**：
- 使用requestAnimationFrame
- 目标帧率：30fps
- Canvas尺寸自适应

### 4.4 统计模块 (Statistics)

**职责**：
- 拍球次数统计
- 训练时长记录
- 频率计算（次/分钟）

## 5. 数据流

```
麦克风输入 → AudioContext → AnalyserNode → 获取数据 → 
    ↓
┌───────────────────┐
│  可视化模块        │  绘制波形和频谱
│  检测模块         │  检测拍球事件
│  统计模块         │  更新统计数据
└───────────────────┘
    ↓
UI 更新
```

## 6. 性能优化策略

1. **动画优化**
   - requestAnimationFrame控制帧率
   - 适当降低绘制频率（30fps）
   - 复用TypedArray缓冲区

2. **内存管理**
   - 组件卸载时销毁AudioContext
   - 及时清理事件监听器
   - 避免内存泄漏

3. **响应式设计**
   - 动态调整Canvas尺寸
   - 移动端简化可视化效果
   - 触摸交互优化
